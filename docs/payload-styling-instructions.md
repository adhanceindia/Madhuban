Styling the Payload Admin Panel
The One File to Edit
app/(payload)/custom.scss — this is your only styling entry point for the admin panel.

Payload v3 exposes its entire theme through CSS custom properties. Override them here.

Available CSS Variables

// custom.scss

:root {
  // ─── Base surfaces ───
  --theme-bg: #f8f9f4;                    // page background
  --theme-elevation-0: #ffffff;           // cards, sidebar
  --theme-elevation-50: #fafbf7;          // hover states
  --theme-elevation-100: #f0f2ec;         // input backgrounds
  --theme-elevation-150: #e5e8df;         // borders
  --theme-elevation-200: #d4d8cd;         // darker borders
  --theme-elevation-250: #b8bfae;         // disabled states
  --theme-elevation-300: #9ca68f;         // placeholder text
  --theme-elevation-350: #6b7a5e;         // secondary text
  --theme-elevation-400: #4a5740;         // muted text
  --theme-elevation-450: #2d3626;         // primary text
  --theme-elevation-500: #1a1f16;         // headings
  --theme-elevation-600: #111410;         // bold text
  --theme-elevation-650: #0a0c08;         // max contrast
  --theme-elevation-700: #050604;

  // ─── Brand colors ───
  --theme-success-500: #386a0e;           // primary actions, buttons
  --theme-success-600: #2f590b;           // hover state
  --theme-error-500: #dc2626;
  --theme-warning-500: #d97706;

  // ─── Typography ───
  --font-body: 'Plus Jakarta Sans', system-ui, sans-serif;
  --font-mono: 'Geist Mono', ui-monospace, monospace;

  // ─── Radius ───
  --style-radius-s: 4px;
  --style-radius-m: 8px;
  --style-radius-l: 12px;
}
What NOT to Touch
File	Why
app/(payload)/layout.tsx	Auto-generated by Payload. Don't add imports.
app/globals.css	Frontend only. Never imported into admin.
tailwind.config.ts content paths	Already excludes (payload). Keep it that way.
@payloadcms/next/css	Payload's internal stylesheet. Can't override directly.
Any file under node_modules/@payloadcms/	Obvious, but worth stating.
Styling Approach — Do's and Don'ts
Do:

Use :root CSS variable overrides in custom.scss
Target Payload's BEM classes (.nav, .dashboard__card-list, .collection-list)
Use --theme-* variables for colors — Payload uses them internally for hover/active states
Inspect the admin panel in DevTools to find class names
Don't:

Don't use @tailwind directives in custom.scss
Don't import Tailwind utilities into the admin
Don't use @apply — it won't work without Tailwind processing
Don't override Payload's JS-injected inline styles with !important spam
Don't add a separate CSS file — custom.scss is the single entry point Payload looks for
Loading Custom Fonts
Add font imports at the top of custom.scss:


@import url('https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@400;500;600;700&display=swap');

:root {
  --font-body: 'Plus Jakarta Sans', system-ui, sans-serif;
}
Targeting Specific Areas

// Sidebar
.nav { background: var(--theme-elevation-0); }
.nav__link--active { border-left: 3px solid var(--theme-success-500); }

// Dashboard cards
.dashboard .card { border-radius: var(--style-radius-l); }

// Collection list views
.collection-list__header { /* table headers */ }
.cell-id { /* ID column cells */ }

// Global header bar
.app-header { border-bottom: 1px solid var(--theme-elevation-150); }
How to Discover Class Names
Open localhost:3000/admin
Right-click any element → Inspect
Payload uses BEM naming: .block__element--modifier
Copy the class, target it in custom.scss
