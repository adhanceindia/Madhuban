## Payload CMS Collections

### Rooms
slug, name, type (Standard/Deluxe/Suite), description (rich text),
price_per_night, capacity, bed_type, room_size,
amenities (array), images (upload multiple), is_active (boolean)

### Bookings
room (relation → Rooms), guest_name, guest_phone, guest_email,
check_in, check_out, guests_count,
payment_method (online/at_reception),
payment_status (pending/paid/failed/refunded),
razorpay_order_id, razorpay_payment_id,
source (website/booking_com/mmt/manual),
status (confirmed/pending/cancelled),
total_amount, created_at

### Inquiries
name, phone, email,
event_type (wedding/birthday/corporate/other),
event_date, guests_count, message,
status (new/contacted/closed), created_at

### BlockedDates
room (relation → Rooms), date, source (ical/manual), ical_uid

### Gallery
image (upload), category (rooms/wedding/events/pool/restaurant),
caption, sort_order

### Reviews
guest_name, rating (1-5), review_text,
source (google/manual), is_published (boolean)

### Content (Site-wide editable content)
tagline, hero_heading, hero_subtext,
wedding_heading, wedding_description,
contact_phone, contact_email, contact_address,
whatsapp_number, instagram_url, facebook_url,
bookingcom_ical_url, mmt_ical_url

## Payload Local API Usage

Use Payload Local API for all server-side data fetching. Never use REST internally.

```typescript
import { getPayload } from 'payload'
import config from '@payload-config'

const payload = await getPayload({ config })

const rooms = await payload.find({
  collection: 'rooms',
  where: { is_active: { equals: true } }
})
```

## Booking Flow

```
Select room + dates
    → GET /api/availability
    → Fill guest details
    → Pay Now → POST /api/payments/order → Razorpay → webhook confirms
    → Pay at Reception → POST /api/bookings → saved as pending
    → Confirmation email via Resend
```

Price: subtotal (nights × price) + 12% GST = total

---

## iCal Sync

```
GET /api/ical/export
    → Confirmed bookings via Payload Local API
    → .ics feed via ical-generator
    → Give this URL to Booking.com + MMT extranets

POST /api/cron/sync-ical (every 30 min via Vercel Cron)
    → Verify CRON_SECRET
    → Pull + parse Booking.com + MMT iCal feeds
    → Upsert into blocked_dates via Payload
    → Update Redis cache
```
